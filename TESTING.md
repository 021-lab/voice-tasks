# Тестирование: Voice Task List PWA

Один последовательный сценарий покрывает все модули и все операции, включая персистентность.

## Предусловия

- `localStorage['vt_env']` не установлен — при открытии появится форма ввода
- Qdrant Cloud: пустые коллекции `tasks` и `log` (создадутся сами при первом `add`)
- iOS Safari или Chromium для первичной отладки
- Микрофон разрешён
- Страница открыта по `https://` (Web Speech API требует HTTPS)

---

## Шаг 1 — Первый запуск

1. Открыть `https://021-lab.github.io/voice-tasks/`
2. Интерфейс рендерится без ошибок в консоли
3. Нажать шестерёнку → textarea с `.env`-полями → вставить credentials → **Сохранить**
4. **Проверить:** `window.__cfg` в консоли содержит `qdrantUrl`, `qdrantKey`, `llmKey`

## Шаг 2 — Создание задачи (`searchModule → storageModule`)

5. Долгое нажатие «Найди/Добавь» → запись стартует → произнести **«добавь позвонить врачу»**
6. **Проверить:** задача `{ title:'позвонить врачу', status:'сегодня', tags:[] }` в списке
7. **Проверить:** Qdrant `tasks` — один point, payload содержит `id` (UUID), `title`, `status:'сегодня'`, `tags:[]`
8. **Проверить:** Qdrant `log` — запись `{ from:'search', to:'storage', data: Patch{op:'add',...} }`

## Шаг 3 — Смена статуса (`receiveModule → extractModule → storageModule`)

9. Долгое нажатие на задачу → произнести **«это фокус»**
10. **Проверить:** статус задачи в списке → «фокус»
11. **Проверить:** `payload.status === 'фокус'` в Qdrant
12. **Проверить:** лог содержит `Patch { op:'update', task:{ id:'...', status:'фокус' } }`

## Шаг 4 — Добавление тега (`receiveModule → extractModule → storageModule`)

13. Долгое нажатие на задачу → **«добавь тег здоровье»**
14. **Проверить:** `payload.tags === ['здоровье']` в Qdrant

## Шаг 5 — Удаление тега (`receiveModule → extractModule → storageModule`)

15. Долгое нажатие на задачу → **«это не здоровье»**
16. **Проверить:** `payload.tags === []` в Qdrant
17. **Проверить:** Patch в логе содержит `_removeTags: ['здоровье']`

## Шаг 6 — Undo (`undoModule`)

18. Нажать кнопку Undo (короткий тап)
19. **Проверить:** `payload.tags === ['здоровье']` (тег восстановлен)
20. **Проверить:** лог содержит `{ from:'undo', to:'storage' }`

## Шаг 7 — Снятие статуса (`receiveModule → extractModule`)

21. Долгое нажатие на задачу → **«это не фокус»**
22. **Проверить:** `payload.status === 'сегодня'` в Qdrant

## Шаг 8 — Поиск / фильтр (`searchModule → storageModule`)

23. Долгое нажатие «Найди/Добавь» → **«добавь купить молоко»** → задача создана
24. Долгое нажатие на «купить молоко» → **«это сделано»**
25. Долгое нажатие «Найди/Добавь» → **«найди сделанные»**
26. **Проверить:** список показывает только «купить молоко»
27. **Проверить:** `SearchQuery { status:'сделано' }` в логе

## Шаг 9 — Голосовой комментарий к логу (`logModule → receiveModule`)

28. Открыть вкладку «Лог»
29. Долгое нажатие на любую запись → **«это была проверка»**
30. **Проверить:** `payload.comment === 'это была проверка'` в Qdrant `log`
31. **Проверить:** текст комментария виден в интерфейсе

## Шаг 10 — Персистентность (Qdrant Cloud)

32. Перезагрузить страницу (`.env` в localStorage — ввод не нужен)
33. **Проверить:** обе задачи подгружаются с правильными данными
34. **Проверить:** вкладка «Лог» содержит все записи с комментарием

## Шаг 11 — Сетевые ошибки

35. DevTools → Network → Offline
36. Долгое нажатие «Найди/Добавь» → **«добавь офлайн задача»**
37. **Проверить:** regex распознал → Qdrant-запрос упал → toast «нет соединения»
38. Долгое нажатие → неизвестная команда → toast «нет сети, LLM недоступен»
39. Включить сеть → та же команда → выполняется через cloudModule

## Матрица покрытия

| Функция | Шаги |
|---|---|
| `settingsModule` — ввод `.env`, сохранение в localStorage | 1–4 |
| `searchModule` + `Patch{op:'add'}` | 5–8 |
| `receiveModule` + `Patch{op:'update', status}` | 9–12 |
| `storageModule` — merge тегов | 13–14 |
| `storageModule` — `_removeTags` | 15–17 |
| `undoModule` — восстановление снапшота | 18–20 |
| `extractModule` — «это не [статус]» → дефолт | 21–22 |
| `storageModule.query(SearchQuery)` | 23–27 |
| `logModule` — голосовой комментарий | 28–31 |
| Qdrant персистентность (reload без ввода) | 32–34 |
| Сетевые ошибки / cloud fallback | 35–39 |

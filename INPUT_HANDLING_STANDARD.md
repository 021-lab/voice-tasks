# Стандарт: единая точка приёма пользовательского ввода

## 1. Маркировка элементов

Каждый интерактивный элемент на странице обязан иметь атрибут:

```html
<li agent-action-id="task-42">...</li>
<button agent-action-id="btn-undo">...</button>
```

`{id}` — уникальный идентификатор элемента в рамках страницы (id задачи, id лога, фиксированное имя кнопки и т.п.).

Элементы без `agent-action-id` не интерактивны и не обрабатываются системой ввода.

## 2. Единый обработчик ввода

Все события исходят из одного модуля — `inputModule`. Запрещены отдельные `addEventListener('click'/'touchstart'/...)`, разбросанные по другим модулям или коду рендера. Привязка слушателей к DOM-элементам должна идти через единственную функцию модуля, например `inputModule.bind(element)`, читающую `agent-action-id` из самого элемента — никаких дополнительных параметров с типом/режимом не передаётся снаружи.

## 3. Распознаваемые действия

Модуль ввода различает только два жеста:

| Жест | Триггер | Действие |
|---|---|---|
| Клик (короткое нажатие, < 200мс) | `touchstart` → `touchend` без сдвига | Подсветить элемент (`flash`) на 1 секунду. Дальше никуда не передаётся. |
| Удержание + голос + отпускание (≥ 200мс) | `touchstart` → запись голоса → `touchend` | Транскрибировать речь и передать результат в интерпретатор. |

Любые другие жесты (свайп, долгое удержание без звука, мультитач и т.п.) игнорируются модулем — не порождают ошибок, просто не дают эффекта.

Жест не должен блокировать скролл страницы: слушатели регистрируются как `passive`, без `preventDefault()`; сдвиг пальца более ~12px отменяет начавшуюся запись и закрывает оверлей.

## 4. Контракт вызова интерпретатора

Единственная точка передачи данных дальше по системе — вызов интерпретатора с фиксированной сигнатурой:

```js
interpreterModule.handle(elementId, input)
```

где:
- `elementId` — строка, взятая из `agent-action-id` элемента, на котором сработал жест;
- `input` — объект вида `{ type: 'click' }` или `{ type: 'transcription', text: string }`.

Интерпретатор сам решает, что означает данный `elementId` и какой `type` пришёл — `inputModule` не содержит знания о бизнес-логике (задача это, лог, кнопка undo и т.д.) и не ветвится по типу элемента.

## 5. Тестовый хук

Помимо реальных жестов, должна быть открытая, задокументированная прямо в HTML-файле (комментарием рядом с кодом) точка вызова интерпретатора в обход событий касания — для тестирующего агента:

```js
// AGENT TEST HOOK: тестирующий агент может вызывать интерпретатор напрямую,
// минуя touch-события, для эмуляции действий пользователя.
// Пример: window.__agentInvoke('task-42', { type: 'click' })
//         window.__agentInvoke('task-42', { type: 'transcription', text: 'перенеси в архив' })
window.__agentInvoke = (elementId, input) => interpreterModule.handle(elementId, input);
```

Эта функция обязана дублировать ровно тот же контракт, что и реальный жест — никакой отдельной логики или валидации сверх той, что есть в `interpreterModule.handle`.

## 6. Что запрещено

- Создавать собственные слушатели событий на элементах вне `inputModule`.
- Ветвить интерпретацию ввода (task/log/кнопка) внутри `inputModule` — это работа интерпретатора.
- Оставлять элементы управления (кнопки, пункты списков) без `agent-action-id`, если они доступны пользователю для взаимодействия.
- Использовать `event.preventDefault()` на `touchstart`, если это блокирует скролл.
